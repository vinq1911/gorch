# embedded/ — gorch inference on microcontrollers

This package contains:

- **`gm1_avr.h`** — a header-only int8 inference kernel (linear + ReLU + argmax)
  that runs on AVR (ATmega2560, ATmega328p) and on the host for reference.
  Weights live in flash via `PROGMEM`; activations stay int8 throughout;
  accumulators are int32; requantization is a fused Q0.31 multiplier + shift
  (matches TFLite's symmetric `MultiplyByQuantizedMultiplier` closely enough
  to round-trip byte-for-byte).
- **`reference.go`** — the Go-side mirror of `gm1_avr.h`. Every arithmetic
  step produces the same byte as the C kernel.
- **`model.go`** — `TinyModel`: a concrete int8 MLP representation, a Go
  reference `Infer`, and emitters for the C headers the firmware consumes.
- **`firmware/`** — a test firmware for ATmega2560 that runs the kernel on
  a model + input generated by the harness, and prints the result over
  UART0. The topology is hard-coded as `LinearI8 → ReLU → LinearI32`; the
  numeric tables come from regenerated headers.
- **`avr_runner/`** — a ~60-line libsimavr host runner that loads an AVR ELF,
  hooks UART0's TX IRQ, and streams every transmitted byte to stdout.
  Needed because the stock `simavr` CLI does not install a UART hook by
  default in at least simavr 1.6.
- **`avr_harness_test.go`** — the test itself: builds the runner, regenerates
  the model+input headers, builds firmware.elf, runs it through the
  simulated AVR, and asserts the firmware's output matches the Go reference
  for several deterministic inputs.

## Running the harness

```
# Dependencies (one-off):
apt-get install gcc-avr avr-libc simavr libsimavr-dev libelf-dev

# Test:
cd embedded && go test -run TestAVRInferenceMatchesReference -v
```

The test `t.Skip`s if `avr-gcc` or `make` aren't on PATH, so CI on machines
without the AVR toolchain still builds and passes pure-Go tests.

## Wire format (`gm1_avr.h`)

A model is a sequence of three layer kinds:

| Kind       | Weights                | Accumulator | Output         |
| ---------- | ---------------------- | ----------- | -------------- |
| LinearI8   | int8, row-major W + i32 bias | int32 | int8 via requant |
| ReLU       | none                   | n/a         | in-place       |
| LinearI32  | int8 W + i32 bias      | int32       | int32 logits   |

The final layer must be `LinearI32` so argmax runs on full-precision
logits.

`M`, `S` per LinearI8 layer are the fused multiplier and right-shift:

    y = saturate(((W·x + B) * M + 2^(S-1)) >> S)

## Footprint on ATmega2560

For a 4→6→3 toy model (what the harness tests):

- Flash: ~1 KB firmware + weights (45 bytes data)
- SRAM: `GM1_MAX_DIM` × 2 int8 ping-pong buffers + `GM1_NUM_CLASSES` × int32 logits
- Inference: well under 1 ms at 16 MHz

A realistic MNIST 784→64→10 MLP fits in ~51 KB of flash and ~1.5 KB of SRAM,
and runs in ~20 ms per digit. See the design discussion in the branch PR
description for the memory math.

## Where gorch training fits in

Not yet wired: a gorch → gm1 exporter that takes a trained `*nn.Linear`
stack, calibrates per-layer activation ranges from a batch, quantizes
weights to int8, computes the fused `M`/`S` constants, and emits the same
header format this harness consumes. Because gorch is darwin-only today,
that exporter will live behind `//go:build darwin`. The harness itself is
host-agnostic: once the exporter exists, the same test path validates it
end-to-end.
